# ABC-анализ товаров собственного производства для food-retail (PostgreSQL)

## Описание бизнес-задачи
В рамках тестового задания требовалось провести **ABC-анализ по марже за сентябрь 2024 года** для товаров собственного производства в магазине с **максимальным объёмом продаж**.

Далее среди товаров **категории A** необходимо было:
- найти позиции, по которым **ухудшилась оборачиваемость относительно августа**;
- выбрать **5 товаров с максимальной рентабельностью продаж за 3 месяца**.

---

# Почему решение реализовано именно так
Для решения задачи был выбран **SQL-first подход**, максимально приближенный к production BI / analytics pipeline.

### Что было сделано
1. **Локально развернута PostgreSQL база данных**
2. Исходный CSV-файл был:
   - очищен
   - приведён к корректному UTF-8 формату
   - нормализован по разделителям
3. Данные были загружены в локальную БД в слой **ODS**
4. Все расчёты выполнены **исключительно средствами SQL**
5. Итоговый результат сохранён:
   - в SQL-витрину
   - в CSV-файлы для передачи на проверку

---

# Описание SQL-скриптов

## `01_create_table.sql`
Создание ODS-таблицы для загрузки исходных данных:
- нормализованные английские названия полей
- типизация под PostgreSQL
- BI-friendly precision для numeric

---

## `02_ABC_top_store.sql`
ABC-анализ по марже для магазина с максимальными продажами за сентябрь.

### Что делает
- определяет топовый магазин
- считает маржу по SKU
- рассчитывает накопительную долю
- присваивает категории:
  - A → до 80%
  - B → 80–95%
  - C → 95–100%

Результат выгружен в:

- `data/abc_top_store.csv`

---

## `03_abc_analysis_solution.sql`
Основной SQL-скрипт решения тестового задания.

### Логика
1. Поиск магазина с максимальными продажами
2. ABC-анализ по марже
3. Отбор товаров категории A
4. Проверка ухудшения оборачиваемости:
   - сентябрь < август
5. Расчёт рентабельности продаж за 3 месяца
6. Отбор **top-5 SKU**

Итоговый результат находится в:

- `data/final_top5_solution.csv`

---

## `04_create_datamart_answer.sql`
Создание итоговой витрины с результатом для BI / dashboarding / проверки.

# 📊 Используемые бизнес-метрики

## Маржа
```sql
sales_k_rub - cogs_k_rub
```

---

## ABC-анализ
На основе накопленной доли маржи:

- A → <= 80%
- B → <= 95%
- C → > 95%

---

## Оборачиваемость
```sql
cogs_k_rub / avg_stock_cost_k_rub
```

Ухудшение:
```sql
turnover_sep < turnover_aug
```

---

## Рентабельность продаж за 3 месяца
```sql
(sum(sales - cogs) / sum(sales)) * 100
```
